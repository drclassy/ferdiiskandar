
# Contributing Guide

## Development Workflow

1. Create a focused branch.
2. Run `npm run lint && npm run typecheck && npm test` before opening a PR.
3. Keep changes scoped and include test coverage for behavior changes.

## Pull Request Requirements

- Clear description of intent and risk.
- Evidence of validation commands.
- Updates to documentation when behavior changes.

## Code Standards

- TypeScript strict mode only.
- Prefer pure utilities and explicit typing.
- Keep component props minimal and stable.
