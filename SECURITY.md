
# Security Policy

## Supported Versions

The default branch and latest release are supported.

## Reporting a Vulnerability

Report security issues privately to maintainers with reproduction details and impact summary. Do not open public issues for unpatched vulnerabilities.

## Disclosure Process

1. Confirm receipt within 72 hours.
2. Triage severity and affected scope.
3. Prepare and validate remediation.
4. Publish a coordinated disclosure after patch availability.

## Dependency and Scanner Exceptions

Any accepted exception must include:
- vulnerability ID
- impact statement
- mitigation strategy
- planned remediation date
