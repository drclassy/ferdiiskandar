## ── FRONT PAGE · WHAT THIS REPOSITORY IS

This repository contains the personal website for **dr Ferdi Iskandar** — a physician, hospital CEO, and founder of **Sentra Artificial Intelligence**.

The project is intentionally structured as a **founder dossier**: an editorial, route-based public profile system that explains identity, work, writing, speaking, CV, collaboration surfaces, and selected systems with clarity and institutional tone.

<table>
<tr>
<td width="33%" valign="top">

### FOUNDER DOSSIER

A publication-grade personal website for presenting founder identity, professional journey, selected works, speaking profile, and public credibility surfaces.

</td>
<td width="33%" valign="top">

### ABBY AI ASSISTANT

A personal AI assistant for helping visitors understand dr Ferdi Iskandar's public profile, work, journey, and collaboration pathways.

</td>
<td width="33%" valign="top">

### EDITORIAL INFRASTRUCTURE

A structured Next.js application with curated public routes, AI provider abstraction, knowledge files, security boundaries, and verification commands.

</td>
</tr>
</table>

The mission is simple: **make the founder's public profile understandable, navigable, credible, and AI-assisted without turning the website into a noisy generic portfolio.**

## ── API REFERENCE · ABBY AND LEGACY CHAT

### `POST /api/abby`

The main Abby conversation endpoint.

**Request body:**

```json
{
  "message": "Siapa dr Ferdi Iskandar?",
  "visitorMode": "public_visitor",
  "history": []
}
```

**Response:**

```json
{
  "reply": "dr Ferdi Iskandar adalah dokter, pemimpin rumah sakit, dan founder yang membangun karya di persimpangan healthcare, leadership, dan artificial intelligence."
}
```

### Operational Behavior

<table>
<tr>
<th align="left">Feature</th>
<th align="left">Implementation</th>
</tr>
<tr>
<td>Rate limiting</td>
<td>Per-IP fixed-window, 20 requests per 60 seconds</td>
</tr>
<tr>
<td>Provider switching</td>
<td>Environment variable <code>AI_PROVIDER</code></td>
</tr>
<tr>
<td>Request validation</td>
<td>Message required, 1–2000 characters</td>
</tr>
<tr>
<td>Timeout protection</td>
<td>28 seconds for Abby, 25 seconds for legacy chat</td>
</tr>
<tr>
<td>Error mapping</td>
<td>Safe, non-leaking upstream error responses</td>
</tr>
<tr>
<td>Output normalization</td>
<td>Plain text, no Markdown</td>
</tr>
<tr>
<td>Cache control</td>
<td><code>Cache-Control: no-store</code> on all API responses</td>
</tr>

## ── CONFIGURATION · ENVIRONMENT CONTRACT

Environment variables are defined in `.env.example`. Copy it to `.env.local` and fill in required values.

```env
# AI Provider selection: "deepseek" default or "openai"
AI_PROVIDER=deepseek

# DeepSeek required when AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
ABBY_MODEL=deepseek-chat

# OpenAI required when AI_PROVIDER=openai
# OPENAI_API_KEY=
# ABBY_MODEL=gpt-4o-mini

# Legacy only, used by /api/chat. Not required for Abby.
NVIDIA_API_KEY=
```

### Runtime Guard

The build process is protected by `scripts/next-runtime-guard.mjs`.

- `pnpm build` is blocked if `next dev` is still active in the same app workspace.
- The guard writes `.next-runtime-lock.json` during dev/build.
- Stop the dev server or remove the lock file when a stale lock blocks execution.
